# Prompt: Explain to a Junior Developer

**Agent-agnostic.** Use with Claude, Copilot, or any LLM.

---

Explain the following code or concept to a junior Java developer (1–2 years experience).

Guidelines:
- Use plain language — avoid jargon unless you define it first
- Use a real-world analogy to introduce the concept
- Walk through the code step by step
- Highlight what could go wrong and how the code handles it (or doesn't)
- Suggest what they should read or explore next

Finish with: "The key thing to remember is..." (one sentence takeaway).
